# VCF Operations Report Downloader

A standalone Python script to download completed PDF reports from **VCF Operations** (formerly VMware Aria Operations / vROps) generated within the last 24 hours.

Report generation can be scheduled normally via the VCF Operations UI, and then this script can be scheduled to download the reports afterwards.


## Features

* **Dual-Configuration Input:** Provide credentials via standard command-line interface (CLI) arguments or abstract them safely into an external configuration file.
* **Security Verification:** Built-in POSIX permission validation flags security risks if your configuration file is exposed to group or world permissions.
* **Chronological Filenames:** Automatically matches internal UUID execution instances to human-readable report definition templates, outputting files using an asset-safe `[Report_Name]_[YYYY-MM-DD_HHMMSS].pdf` pattern.
* **Efficient Lookup Cache:** Pre-fetches report definitions in a single call to build a lightweight in-memory lookup table, avoiding API bloat and reducing network round-trips.
* **Future-Proof Time Handling:** Leverages explicit, timezone-aware UTC datetime tracking compliant with Python 3.12+ specifications.


## Prerequisites

* **Python Runtime:** Python 3.8 or newer (Fully compliant with Python 3.12+).
* **Dependencies:** The script requires the third-party `requests` library. Install it globally or within your Python environment:

```bash
pip install requests

```


## Configuration File Setup

To separate sensitive credentials from process scheduling logic, it is highly recommended to store environment data in a dedicated configuration file (e.g., `/etc/vcf_ops.conf`).

Create an INI-style file structured as follows:

```ini
[vcf-ops]
host = vcf-ops.yourdomain.local
user = automation_svc
password = YourSecurePasswordHere
authsource = internal
outdir = /mnt/nfs_reports_share

```

### Locking Down Permissions (Crucial)

Because this file contains cleartext administrative passwords, enforce strict ownership restrictions. The script will throw a security warning at runtime if permissions are not set correctly:

```bash
chmod 600 /etc/vcf_ops.conf

```


## Command Line Usage

```bash
python vcf_report_downloader.py [options]

```

### Argument Reference

| Flag | Long Flag | Description | Default |
| --- | --- | --- | --- |
| `-c` | `--config` | Path to an optional INI configuration file containing defaults. | None |
| `-H` | `--host` | VCF Operations fully qualified domain name (FQDN) or IP. | None |
| `-u` | `--user` | VCF Operations automation service account user. | None |
| `-p` | `--password` | VCF Operations account password. | None |
| `-a` | `--authsource` | Identity provider engine name for authentication. | `internal` |
| `-o` | `--outdir` | Direct destination folder path where PDFs will be stored. | None |
| `-U` | `--unsafe` | Skips SSL certificate validation (Lab/Self-Signed friendly). | `False` |

> **Note:** CLI flags will always take absolute precedence over parameters defined inside an attached configuration file.


## Examples

### 1. Execution via Locked Config File (Best Practice for Cron)

Perfect for automated daily cron utilities or task schedulers:

```bash
python vcf_report_downloader.py -c /etc/vcf_ops.conf -U

```

### 2. Full CLI Execution (No Config File)

Provide all fields dynamically at runtime:

```bash
python vcf_report_downloader.py -H vcf-ops.local -u administrator -p "Pass123" -o /tmp/downloads -U

```

### 3. Dynamic Override Configuration

Load baseline environment configurations from the INI file, but manually redirect the payload files to an isolated sandbox on the fly:

```bash
python vcf_report_downloader.py -c /etc/vcf_ops.conf -o /tmp/temporary_report_dump -U

```


## Automation Placement (Cron Job Example)

To configure this automation to parse and gather generated reports every night at midnight, append the following line to your crontab environment (`crontab -e`):

```text
0 0 * * * /usr/bin/python3 /opt/scripts/vcf_report_downloader.py -c /etc/vcf_ops.conf -U > /var/log/vcf_report_sync.log 2>&1

```


## Important Security Note on Scheduled Reports

The VCF Operations Suite-API applies contextual access scoping restrictions to generated report datasets:

* **Scope Rule:** The `/suite-api/api/reports` execution register natively outputs report instances *exclusively* generated by the user credentials active in the session request.
* **Action Required:** Ensure that the internal VCF Operations Scheduler engine generates your daily system reports under the **exact same account** assigned to the `user` block in this script's config. If the scheduler builds reports under a different account, this script will complete with a success message but return 0 downloaded payloads.
